Deux mini examens similaires corriges

Ces sujets sont construits dans le meme esprit que le module: definitions, comparaisons, interet des mecanismes Oracle et reconnaissance des commandes ou options importantes.

Examen similaire A

  • Q1. Citer deux privileges systeme et expliquer leur utilite.
    Resultat: Exemple CREATE SESSION pour se connecter, CREATE TABLE pour creer des tables.
  • Q2. A quoi sert un role ?
    Resultat: A factoriser plusieurs privileges sous un nom unique pour simplifier l’administration.
  • Q3. Donner deux parametres de profil lies au mot de passe.
    Resultat: FAILED_LOGIN_ATTEMPTS, PASSWORD_LIFE_TIME.
  • Q4. Difference entre authentification base et OS ?
    Resultat: Dans le premier cas Oracle verifie le mot de passe; dans le second, Oracle s’appuie sur le systeme d’exploitation.

Examen similaire B

  • Q1. Difference entre restauration et recuperation.
    Resultat: Restauration = recopier un backup; recuperation = reappliquer les modifications necessaires.
  • Q2. Citer trois categories de panne.
    Resultat: Panne d’instance, panne physique, erreur utilisateur.
  • Q3. Utilite de la FRA.
    Resultat: Zone reservee aux fichiers utiles au recovery et a la gestion des sauvegardes.
  • Q4. Difference entre recuperation complete et incomplete.
    Resultat: Complete vise le plus recent possible; incomplete vise un point precis.

Examen similaire C

  • Q1. C’est quoi RMAN ?
    Resultat: L’outil Oracle integre de sauvegarde et recuperation.
  • Q2. A quoi sert TARGET ?
    Resultat: A designer la base cible.
  • Q3. Pourquoi utiliser SYSBACKUP ?
    Resultat: Pour separer les responsabilites de sauvegarde des autres droits sur les donnees.
  • Q4. A quoi sert SHOW ALL ?
    Resultat: A afficher les parametres persistants RMAN.

Examen similaire D

  • Q1. Control file ou recovery catalog: que stocke le deuxieme de plus utile ?
    Resultat: Historique plus riche, scripts stockes, gestion de plusieurs cibles et rapports plus complets.
  • Q2. Le recovery catalog est-il obligatoire ?
    Resultat: Non, RMAN peut fonctionner avec le control file seul.
  • Q3. Pourquoi prevoir sa taille ?
    Resultat: Parce que le volume de metadonnees depend du nombre de bases et de sauvegardes suivies.

Examen similaire E

  • Q1. Qu’est-ce qu’un incremental level 0 ?
    Resultat: Une sauvegarde de base equivalente a une sauvegarde complete de reference.
  • Q2. Difference cumulative / differentielle ?
    Resultat: Cumulative depuis level 0, differentielle depuis le dernier incremental.
  • Q3. Image copy vs backup set ?
    Resultat: La premiere est plus simple a restaurer, le second economise l’espace.

Examen similaire F

  • Q1. Role de RESTORE et de RECOVER.
    Resultat: RESTORE remet les fichiers, RECOVER rejoue les changements.
  • Q2. Quand utilise-t-on OPEN RESETLOGS ?
    Resultat: Souvent apres recuperation incomplete ou a partir d’un control file de sauvegarde.
  • Q3. Pourquoi le DBID peut-il etre necessaire ?
    Resultat: Pour identifier la base dans certains scenarios de restauration, notamment spfile/control file.

Examen similaire G — Flashback

  • Q1. Qu'est-ce que Flashback Query ? Quelle clause SQL utilise-t-on ?
    Resultat: Interroge les donnees telles qu'elles existaient a un instant passe. Clause : AS OF TIMESTAMP ou AS OF SCN.
  • Q2. Quel est le prerequis obligatoire pour FLASHBACK TABLE ?
    Resultat: Activer ROW MOVEMENT : ALTER TABLE t ENABLE ROW MOVEMENT;
  • Q3. Comment recuperer une table supprimee accidentellement ?
    Resultat: FLASHBACK TABLE nom TO BEFORE DROP; (a condition que la corbeille ne soit pas purgee).
  • Q4. Que fait DROP TABLE t PURGE par rapport a DROP TABLE t ?
    Resultat: PURGE supprime definitivement la table sans passer par la corbeille, rendant Flashback Drop impossible.

Examen similaire H — Flashback Database

  • Q1. Quelles sont les conditions pour activer Flashback Database ?
    Resultat: Mode ARCHIVELOG actif, FRA configuree (db_recovery_file_dest et taille), puis ALTER DATABASE FLASHBACK ON en mode MOUNT.
  • Q2. Quelle commande ouvre la base apres un FLASHBACK DATABASE ?
    Resultat: ALTER DATABASE OPEN RESETLOGS; — RESETLOGS est obligatoire.
  • Q3. Difference entre Restore Point normal et Guaranteed Restore Point ?
    Resultat: Normal: valide tant que les flashback logs sont disponibles. Guaranteed: Oracle garantit de ne pas ecraser les logs necessaires jusqu'a suppression explicite du restore point.
  • Q4. Quel vue montre si Flashback Database est active ?
    Resultat: SELECT flashback_on FROM v$database;

Examen similaire I

  • Q1. Quel est le role d’un alias TNS ?
    Resultat: Fournir un nom simple qui encapsule les details de connexion.
  • Q2. Quelle commande utilise-t-on pour verifier l’etat du listener ?
    Resultat: lsnrctl status.
  • Q3. Qui fait quoi entre alias et listener ?
    Resultat: L’alias indique ou aller; le listener recoit la connexion et la transfere a l’instance.
