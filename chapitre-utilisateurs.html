Utilisateurs, privileges, roles et profils

Toute la securite logique Oracle avec la syntaxe exacte de chaque commande. Ce chapitre couvre l'integralite des points du cours.

Notions cles Utilisateurs Privileges Roles Profils Vues

Notions cles

  • Utilisateur — identite Oracle. Possede un schema (espace d'objets) et des droits. Doit avoir au minimum CREATE SESSION pour se connecter.
  • Quota — limite l'espace qu'un utilisateur peut consommer dans un tablespace.
  • Privilege systeme — autorise une action globale sur la base (ex: creer une table).
  • Privilege objet — autorise une action sur un objet precis d'un schema (ex: SELECT sur une table).
  • Role — regroupement nomme de privileges pour simplifier l'administration.
  • Profil — ensemble de limites sur les mots de passe et les ressources systeme.

Differences a ne pas confondre

  • Privilege systeme = action generale sur la base.
  • Privilege objet = action sur un objet precis.
  • Role = conteneur de privileges (facilite l'administration).
  • Profil = limite les mots de passe ET les ressources.
  • WITH GRANT OPTION = transmet un privilege objet.
  • WITH ADMIN OPTION = transmet un privilege systeme ou role.
  • SYSBACKUP = role pour RMAN (pas un privilege).

1. Gestion des utilisateurs

Elements definis a la creation

  • Nom + methode d'authentification (base de donnees ou OS).
  • Tablespace par defaut (stockage des objets permanents).
  • Tablespace temporaire (tri, jointures, operations temporaires).
  • Quotas sur les tablespaces (espace autorise).
  • Privileges et roles initiaux.

Authentification

  • Par la base — Oracle verifie l'identifiant et le mot de passe. Mode le plus courant.
  • Par l'OS — Oracle fait confiance a l'authentification du systeme d'exploitation. Mot-cle EXTERNALLY.
CREATE USER — syntaxe complete 
CREATE USER app_user IDENTIFIED BY mon_password
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp
  QUOTA 50M ON users
  ACCOUNT UNLOCK;

-- Authentification par l'OS
CREATE USER user_name IDENTIFIED EXTERNALLY;

-- Creation minimale (tablespaces et quota par defaut)
CREATE USER user_name IDENTIFIED BY password;
ALTER USER — toutes les variantes 
-- Changer le mot de passe
ALTER USER app_user IDENTIFIED BY nouveau_password;

-- Verrouiller le compte (connexion impossible)
ALTER USER app_user ACCOUNT LOCK;

-- Deverrouiller le compte
ALTER USER app_user ACCOUNT UNLOCK;

-- Forcer le changement de mot de passe a la prochaine connexion
ALTER USER app_user PASSWORD EXPIRE;

-- Modifier le tablespace par defaut
ALTER USER app_user DEFAULT TABLESPACE data_tbs;

-- Modifier le tablespace temporaire
ALTER USER app_user TEMPORARY TABLESPACE temp2;

-- Modifier les quotas
ALTER USER app_user QUOTA 100M ON users;
ALTER USER app_user QUOTA UNLIMITED ON users;

-- Supprimer le quota (ramene a 0)
ALTER USER app_user QUOTA 0 ON users;

-- Affecter un profil
ALTER USER app_user PROFILE profil_app;
DROP USER 
-- Echoue si l'utilisateur possede des objets
DROP USER app_user;

-- Supprime l'utilisateur ET tous ses objets (tables, vues, sequences...)
DROP USER app_user CASCADE;
Exam: DROP USER CASCADE supprime aussi tout le schema. Sans CASCADE, Oracle refuse si le schema n'est pas vide.
Verification via dictionnaire 
SELECT username, account_status, default_tablespace, profile, expiry_date
  FROM dba_users
  ORDER BY username;

SELECT username, tablespace_name, bytes, max_bytes
  FROM dba_ts_quotas
  WHERE username = 'APP_USER';

2. Gestion des privileges

TypeDefinitionExemples
Privilege systeme Action generale sur la base ou sur des categories d'objets. CREATE SESSION, CREATE TABLE, CREATE VIEW, ALTER SYSTEM, GRANT UNLIMITED TABLESPACE
Privilege objet Action sur un objet precis d'un schema. SELECT, INSERT, UPDATE, DELETE, EXECUTE sur une table/vue/procedure
GRANT — privileges systeme 
GRANT CREATE SESSION TO app_user;
GRANT CREATE TABLE TO app_user;
GRANT CREATE VIEW TO app_user;
GRANT CREATE SEQUENCE TO app_user;
GRANT CREATE PROCEDURE TO app_user;
GRANT UNLIMITED TABLESPACE TO app_user;

-- Avec droit de transmettre (WITH ADMIN OPTION)
GRANT CREATE TABLE TO app_user WITH ADMIN OPTION;
GRANT — privileges objet 
-- Donner plusieurs privileges sur une table
GRANT SELECT, INSERT, UPDATE, DELETE
  ON scott.emp
  TO app_user;

-- Privilege sur certaines colonnes seulement
GRANT UPDATE (sal, comm)
  ON scott.emp
  TO app_user;

-- Avec droit de transmettre le privilege (WITH GRANT OPTION)
GRANT SELECT ON scott.emp TO app_user WITH GRANT OPTION;

-- Donner a tous les utilisateurs
GRANT SELECT ON scott.emp TO PUBLIC;
REVOKE — retirer des privileges 
-- Retirer un privilege systeme
REVOKE CREATE TABLE FROM app_user;

-- Retirer un privilege objet
REVOKE SELECT ON scott.emp FROM app_user;

-- Retirer plusieurs privileges objet
REVOKE INSERT, UPDATE, DELETE ON scott.emp FROM app_user;
Verification des privileges 
-- Privileges systeme accordes
SELECT grantee, privilege, admin_option
  FROM dba_sys_privs
  WHERE grantee = 'APP_USER';

-- Privileges objet accordes
SELECT grantee, owner, table_name, privilege, grantable
  FROM dba_tab_privs
  WHERE grantee = 'APP_USER';

-- Privileges actifs dans la session courante
SELECT * FROM session_privs;
WITH GRANT OPTION vs WITH ADMIN OPTION: WITH GRANT OPTION s'applique aux privileges objet (transmet le droit sur l'objet). WITH ADMIN OPTION s'applique aux privileges systeme et aux roles (transmet le droit de gestion).

3. Gestion des roles

Un role regroupe des privileges sous un nom. On attribue le role plutot que chaque privilege individuellement. Un seul REVOKE retire tous les privileges du role.

Cycle complet d'un role 
-- Creer un role
CREATE ROLE role_compta;

-- Donner des privileges au role
GRANT CREATE SESSION TO role_compta;
GRANT SELECT, INSERT, UPDATE, DELETE
  ON finance.factures
  TO role_compta;

-- Donner le role a un utilisateur
GRANT role_compta TO app_user;

-- Donner le role avec possibilite de le transmettre
GRANT role_compta TO app_user WITH ADMIN OPTION;

-- Retirer le role d'un utilisateur
REVOKE role_compta FROM app_user;

-- Supprimer le role
DROP ROLE role_compta;
Verification des roles 
-- Roles accordes aux utilisateurs
SELECT grantee, granted_role, default_role, admin_option
  FROM dba_role_privs
  WHERE grantee = 'APP_USER';

-- Roles actifs dans la session courante
SELECT * FROM session_roles;

Roles standards a connaitre

RoleContenu
CONNECTPrincipalement CREATE SESSION. Historiquement contenait plus de privileges.
RESOURCEPermet la creation d'objets: tables, sequences, procedures, etc.
DBARole puissant, reserve aux administrateurs. Contient la majorite des privileges systeme.
SYSBACKUPRole recommande pour RMAN: couvre backup et recovery sans donner acces aux donnees applicatives.
Exam: SYSBACKUP est un ROLE (pas un privilege). Il est prefere au role DBA pour RMAN car il respecte le principe du moindre privilege.

4. Gestion des profils

Un profil centralise les limites de mots de passe et de ressources. Chaque utilisateur a un profil (DEFAULT par defaut). Les limites de ressources ne s'appliquent que si le parametre RESOURCE_LIMIT = TRUE.

Parametres de mot de passe

ParametreRole
FAILED_LOGIN_ATTEMPTSNombre max de tentatives echouees avant verrouillage du compte.
PASSWORD_LOCK_TIMEDuree de verrouillage apres les echecs (en jours, fractions acceptees).
PASSWORD_LIFE_TIMEDuree de validite du mot de passe (en jours).
PASSWORD_GRACE_TIMEPeriode de grace apres expiration: l'utilisateur peut encore se connecter mais doit changer son mot de passe.
PASSWORD_REUSE_TIMEDelai minimal avant de pouvoir reutiliser un ancien mot de passe (en jours).
PASSWORD_REUSE_MAXNombre de changements requis avant de reutiliser un mot de passe.

Parametres de ressources

ParametreRole
SESSIONS_PER_USERNombre max de sessions simultanees autorisees pour cet utilisateur.
CONNECT_TIMEDuree maximale d'une session connectee (en minutes).
IDLE_TIMEDuree maximale d'inactivite avant deconnexion automatique (en minutes).
LOGICAL_READS_PER_SESSIONNombre max de lectures logiques par session (blocs lus).
PRIVATE_SGAEspace SGA prive alloue a chaque session (mode serveurs partages).
CREATE PROFILE — syntaxe complete 
CREATE PROFILE profil_app LIMIT
  FAILED_LOGIN_ATTEMPTS  3
  PASSWORD_LOCK_TIME     1
  PASSWORD_LIFE_TIME     90
  PASSWORD_GRACE_TIME    7
  SESSIONS_PER_USER      2
  CONNECT_TIME           60
  IDLE_TIME              15
  LOGICAL_READS_PER_SESSION 1000000;

-- Affecter le profil a un utilisateur
ALTER USER app_user PROFILE profil_app;

-- Remettre le profil par defaut
ALTER USER app_user PROFILE DEFAULT;
Verification des profils 
-- Voir les limites d'un profil
SELECT profile, resource_name, limit
  FROM dba_profiles
  WHERE profile = 'PROFIL_APP'
  ORDER BY resource_name;

-- Voir quel profil est affecte a un utilisateur
SELECT username, profile
  FROM dba_users
  WHERE username = 'APP_USER';
Exam: Distinguer PASSWORD_LOCK_TIME (duree de verrouillage apres echecs) de PASSWORD_LIFE_TIME (duree de validite normale du mot de passe). Ce sont deux parametres independants souvent confondus.
