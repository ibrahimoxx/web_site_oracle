Listener Oracle et configuration des alias

Cette partie explique comment une connexion reseau Oracle atteint l’instance: le client utilise un alias TNS, le listener ecoute sur un port, puis redirige vers la base cible.

Listener Fichiers LSNRCTL Commandes

1. Qu’est-ce qu’un listener

  • Processus serveur independant de la base.
  • Ecoute les demandes de connexion entrantes.
  • Ecoute en general sur TCP/IP, souvent port 1521.
  • Dirige la demande vers l’instance appropriee.

2. Fichiers de configuration

  • listener.ora: configuration du listener cote serveur.
  • tnsnames.ora: alias de connexion cote client.
  • sqlnet.ora: parametres reseau globaux.

3. Elements de listener.ora

  • PROTOCOL: protocole de transport.
  • HOST: machine serveur.
  • PORT: port d’ecoute.
  • SID_NAME ou service concerne.

4. Alias TNS

Un alias TNS remplace une description longue de connexion par un nom simple. Le client appelle cet alias, qui encapsule hote, port et service_name.

5. Comment tout fonctionne ensemble

  1. Le client lance une connexion vers un alias.
  2. L’alias est resolu en hote, port et service.
  3. Le listener recoit la demande.
  4. Le listener transfere la connexion a l’instance cible.

6. Commandes LSNRCTL a connaitre

  • lsnrctl start
  • lsnrctl stop
  • lsnrctl status
  • lsnrctl reload
  • lsnrctl services
Revision syntaxe
Point fort revision: cette partie est courte, donc les syntaxes de listener.ora, de l'alias TNS et de lsnrctl doivent etre retenues presque mot pour mot.

7. Commandes et blocs du cours a memoriser

Exemple listener.ora

LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)
                 (HOST = serveur.domain.com)
                 (PORT = 1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = ORCL)
      (ORACLE_HOME = /u01/app/oracle/product/19c)
      (SID_NAME = ORCL)
    )
  )

Commandes LSNRCTL

lsnrctl start
lsnrctl stop
lsnrctl status
lsnrctl reload
lsnrctl services
lsnrctl help

Exemple tnsnames.ora

ORCL_PROD =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)
               (HOST = prod-server.domain.com)
               (PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = ORCL)
    )
  )

Utilisation cote client

sqlplus user/password@ORCL_PROD

Resume flash

L’alias dit ou aller, le listener accueille la demande, puis l’instance prend la main. C’est la formule la plus simple a retenir pour expliquer le mecanisme.
