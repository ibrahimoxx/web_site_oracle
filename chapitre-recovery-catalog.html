Recovery Catalog RMAN

Quand et pourquoi utiliser un catalog, comparaison avec le control file, et toutes les commandes de creation et de maintenance.

1. Definition

Le recovery catalog est une base de donnees Oracle distincte dans laquelle RMAN stocke ses metadonnees: historique des sauvegardes, structure des bases enregistrees, scripts RMAN.

  • Il n'est pas obligatoire pour que RMAN fonctionne.
  • Sans lui, RMAN utilise le control file de la base cible (historique limite).
  • Avec lui, on gagne en richesse d'historique et en flexibilite.

2. Interet principal

  • Historique des sauvegardes plus long que ce que le control file peut stocker.
  • Stocker et reutiliser des scripts RMAN (partageable entre bases).
  • Gerer plusieurs bases cibles depuis un meme referentiel.
  • Produire des rapports personnalises.
  • Supporter des options comme KEEP FOREVER.
  • Survie au cas ou le control file de la cible est perdu.

3. Control file vs Recovery Catalog

ModeAvantagesLimites
Control file seulSimple, pas de base supplementaire a administrer.Historique limite (taille du control file). Moins de fonctionnalites.
Recovery CatalogHistorique complet, scripts stockes, multi-bases, rapports avances, KEEP FOREVER.Necessite une base supplementaire et son administration.

4. Creation du catalog — etapes completes

  1. Preparer une base Oracle qui va heberger le catalog (base distincte).
  2. Creer un utilisateur proprietaire avec un tablespace dedie et suffisamment d'espace.
  3. Accorder le role RECOVERY_CATALOG_OWNER.
  4. Se connecter a RMAN avec la cible ET le catalog.
  5. Creer le catalog.
  6. Enregistrer chaque base cible.

5. Commandes de creation et maintenance

Creer l'utilisateur et le role 
-- Sur la base du catalog (pas la base cible)
CREATE USER rcat_owner IDENTIFIED BY password
  DEFAULT TABLESPACE rcat_ts
  QUOTA UNLIMITED ON rcat_ts;

-- Accorder le role necessaire
GRANT RECOVERY_CATALOG_OWNER TO rcat_owner;
GRANT CREATE SESSION TO rcat_owner;
Connexion RMAN avec le catalog 
-- Connexion avec base cible ET catalog
rman TARGET / CATALOG rcat_owner/password@catdb

-- Ou depuis l'invite RMAN
rman
CONNECT TARGET /
CONNECT CATALOG rcat_owner/password@catdb
Creer le catalog et enregistrer la base 
-- Creer le schema du catalog (a faire une seule fois)
CREATE CATALOG;

-- Enregistrer la base cible dans le catalog
REGISTER DATABASE;

-- Verifier l'enregistrement
LIST DB_UNIQUE_NAME ALL;
Maintenance du catalog 
-- Synchroniser le catalog avec le control file de la cible
RESYNC CATALOG;

-- Synchroniser manuellement depuis un control file specifique
RESYNC CATALOG FROM CONTROLFILECOPY '/backup/cf_backup.ctl';

-- Mettre a jour le catalog (apres mise a jour RMAN)
UPGRADE CATALOG;

-- Deregistrer une base (la retirer du catalog)
UNREGISTER DATABASE;
A retenir: Le recovery catalog stocke des metadonnees RMAN, pas les donnees applicatives. C'est un depot de references, pas une sauvegarde de la base elle-meme.
