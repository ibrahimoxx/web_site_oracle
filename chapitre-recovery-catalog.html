Recovery Catalog RMAN

Cette partie du cours explique quand RMAN peut se contenter du control file et quand il devient preferable d’ajouter un catalogue de recuperation externe.

Concepts Creation Gestion Commandes

1. Definition

Le recovery catalog est une base de donnees distincte dans laquelle RMAN stocke des metadonnees sur les sauvegardes et les restaurations.

  • Il n’est pas obligatoire pour que RMAN fonctionne.
  • Sans lui, RMAN peut utiliser le control file de la base cible.
  • Avec lui, on gagne en richesse d’historique et en flexibilite d’administration.

2. Interet principal

  • Conserver l’historique des sauvegardes plus largement.
  • Stocker et reutiliser des scripts RMAN.
  • Produire des rapports personnalises.
  • Gerer plusieurs bases cibles depuis un meme referentiel.
  • Supporter des options comme KEEP FOREVER.

3. Control file vs Recovery Catalog

Mode Forces Limites
Control file Simple, direct, administration plus legere Historique et services plus limites
Recovery Catalog Historisation, scripts stockes, multi-cibles, rapports Necessite une base distincte et plus d’administration

4. Creation logique

  1. Preparer la base qui accueillera le catalogue.
  2. Creer le proprietaire du catalogue avec le privilege RECOVERY_CATALOG_OWNER.
  3. Creer ensuite le catalogue lui-meme.

Le cours souligne qu’il faut aussi dimensionner l’espace selon le nombre de bases et de sauvegardes suivies.

Point fort revision: pour ce chapitre, l'examen demande souvent a quoi sert le catalog et quelles sont les operations de gestion autour de lui.

5. Gestion logique du catalog

  • Enregistrer une base cible dans le catalog.
  • Retirer une base cible du catalog.
  • Resynchroniser les metadonnees entre control file et catalog.
  • Cataloguer des fichiers de sauvegarde additionnels.
  • Dans le cas du VPC, ajouter ou retirer l'acces avec des commandes de type GRANT et REVOKE.
Revision syntaxe

6. Commandes du cours a memoriser

Preparation du proprietaire

Le support impose un owner avec un tablespace dedie et le privilege du catalog.

CREATE USER rman IDENTIFIED BY motdepasse
DEFAULT TABLESPACE rman_cat_ts
QUOTA UNLIMITED ON rman_cat_ts;

GRANT RECOVERY_CATALOG_OWNER TO rman;

Creation du catalog

Commande explicitement citee dans le support.

CREATE CATALOG;

Gestion courante

Le cours cite ces operations de gestion autour du recovery catalog.

REGISTER DATABASE;
UNREGISTER DATABASE;
RESYNC CATALOG;

VPC et suppression

Le support mentionne GRANT/REVOKE pour VPC et la suppression avec DROP CATALOG.

GRANT ... TO vpc_owner;
REVOKE ... FROM vpc_owner;
DROP CATALOG;

Resume flash

Retenir surtout ceci: le recovery catalog n’est pas une sauvegarde de la base applicative. C’est un depot de metadonnees RMAN, utile pour mieux piloter, historiser et mutualiser les operations.
