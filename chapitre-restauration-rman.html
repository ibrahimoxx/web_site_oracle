Restauration et recuperation RMAN — 7 scenarios

Chaque scenario de restauration avec la sequence complete de commandes, les prerequis et les points d'attention exam.

Principe Tablespace SYSTEM/UNDO Base complete PITR date PITR SCN Control file SPFILE Maintenance

Principe general — Restore puis Recover

EtapeCommandeCe qui se passe
1. RestaurerRESTORERMAN recopie le fichier de sauvegarde vers l'emplacement cible. Fichier present mais potentiellement ancien.
2. RecupererRECOVERRMAN applique les archived redo logs et les redo logs en ligne pour avancer le fichier jusqu'a l'etat coherent le plus recent.
3. OuvrirALTER DATABASE OPEN ou OPEN RESETLOGSRecuperation complete → OPEN. Recuperation incomplete (PITR) → OPEN RESETLOGS (cree une nouvelle incarnation).
Regle absolue: Apres toute recuperation incomplete (PITR, toute utilisation de SET UNTIL), il est OBLIGATOIRE d'utiliser ALTER DATABASE OPEN RESETLOGS. Utiliser ALTER DATABASE OPEN seul causerait une incoherence.

Scenario 1 — Recuperation d'un tablespace non critique

Le tablespace n'est pas SYSTEM ni UNDO. La base peut rester ouverte. Le tablespace est mis OFFLINE, restaure, recupere, puis remis ONLINE.

Sequence complete 
-- 1. Mettre le tablespace offline immediatement
ALTER TABLESPACE users OFFLINE IMMEDIATE;

-- 2. Restaurer et recuperer (base ouverte, RMAN se connecte normalement)
rman TARGET /

RESTORE TABLESPACE users;
RECOVER TABLESPACE users;

-- 3. Remettre le tablespace en ligne
ALTER TABLESPACE users ONLINE;
Remarque: Si un datafile specifique est endommage et non tout le tablespace, utiliser RESTORE DATAFILE n; RECOVER DATAFILE n; avec le fichier mis OFFLINE au prealable.

Scenario 2 — Recuperation du tablespace SYSTEM ou UNDO

Le tablespace SYSTEM ou UNDO ne peut pas etre mis offline avec la base ouverte. La base doit etre en mode MOUNT pour la restauration.

Sequence complete 
-- 1. Arreter la base proprement
SHUTDOWN ABORT;   -- ou IMMEDIATE si la base repond encore

-- 2. Monter la base (pas OPEN)
STARTUP MOUNT;

-- 3. Restaurer et recuperer
rman TARGET /

RESTORE TABLESPACE system;
RECOVER TABLESPACE system;
-- (ou TABLESPACE undotbs1 selon le cas)

-- 4. Ouvrir la base
ALTER DATABASE OPEN;

Scenario 3 — Recuperation complete de la base

Tous les datafiles sont endommages ou perdus. Recuperation complete jusqu'au dernier commit possible.

Sequence complete 
-- 1. Arreter la base
SHUTDOWN ABORT;

-- 2. Monter la base
STARTUP MOUNT;

-- 3. Restaurer puis recuperer toute la base
rman TARGET /

RESTORE DATABASE;
RECOVER DATABASE;

-- 4. Ouvrir la base (recuperation complete → pas de RESETLOGS)
ALTER DATABASE OPEN;

Scenario 4 — Recuperation incomplete PITR par date (timestamp)

Revenir a un point dans le temps specifique, exprime par une date. Exemple : annuler une suppression accidentelle intervenue a 14h30.

Sequence complete 
-- 1. Arreter la base
SHUTDOWN IMMEDIATE;

-- 2. Monter la base
STARTUP MOUNT;

-- 3. RMAN : definir le point cible et recuperer
rman TARGET /

RUN {
  SET UNTIL TIME "TO_DATE('2026-06-06 14:30:00','YYYY-MM-DD HH24:MI:SS')";
  RESTORE DATABASE;
  RECOVER DATABASE;
}

-- 4. Ouvrir avec RESETLOGS (OBLIGATOIRE apres PITR)
ALTER DATABASE OPEN RESETLOGS;
Exam: Apres toute recuperation avec SET UNTIL, l'ouverture se fait uniquement avec OPEN RESETLOGS. Jamais avec OPEN seul.

Scenario 5 — Recuperation incomplete PITR par SCN

Revenir a un SCN (System Change Number) specifique. Plus precis qu'une date. Utile quand le SCN de l'evenement est connu (ex: via Flashback Query).

Sequence complete 
-- 1. Arreter la base
SHUTDOWN IMMEDIATE;

-- 2. Monter la base
STARTUP MOUNT;

-- 3. RMAN : definir le SCN cible et recuperer
rman TARGET /

RUN {
  SET UNTIL SCN 1500000;
  RESTORE DATABASE;
  RECOVER DATABASE;
}

-- 4. Ouvrir avec RESETLOGS
ALTER DATABASE OPEN RESETLOGS;
Trouver un SCN utile avant la recuperation 
-- SCN actuel de la base
SELECT current_scn FROM v$database;

-- Via Flashback Query pour verifier l'etat a un SCN
SELECT * FROM employees AS OF SCN 1499000;

Scenario 6 — Recuperation du control file depuis un autobackup

Tous les control files sont perdus. RMAN peut les restaurer depuis l'autobackup si CONFIGURE CONTROLFILE AUTOBACKUP ON etait actif.

Sequence complete 
-- 1. La base ne peut pas demarrer normalement : lancer NOMOUNT
STARTUP NOMOUNT;

-- 2. Depuis RMAN, restaurer le control file
rman TARGET /

-- Restauration automatique (RMAN cherche dans la FRA)
RESTORE CONTROLFILE FROM AUTOBACKUP;

-- Ou specifier le chemin exact si l'autobackup est dans un repertoire precis
RESTORE CONTROLFILE FROM AUTOBACKUP
  RECOVERY AREA '/u01/app/oracle/fra'
  DB_NAME 'ORCL';

-- 3. Monter la base (maintenant que les control files sont la)
ALTER DATABASE MOUNT;

-- 4. Restaurer et recuperer la base
RESTORE DATABASE;
RECOVER DATABASE;

-- 5. Ouvrir avec RESETLOGS
ALTER DATABASE OPEN RESETLOGS;
Ordre des etapes: NOMOUNT → RESTORE CONTROLFILE → MOUNT → RESTORE DATABASE → RECOVER DATABASE → OPEN RESETLOGS. Ne jamais inverser NOMOUNT et MOUNT.

Scenario 7 — Recuperation du SPFILE depuis un autobackup

Le fichier de parametres SPFILE est perdu. La base ne peut plus demarrer. RMAN peut le restaurer si l'autobackup du control file etait actif (il inclut aussi le SPFILE).

Sequence complete 
-- 1. Demarrer en NOMOUNT avec un PFILE minimal (ou le PFILE par defaut)
-- Oracle cree un PFILE minimal si aucun SPFILE ni PFILE n'est trouve
STARTUP NOMOUNT;

-- 2. Depuis RMAN, restaurer le SPFILE
rman TARGET /

RESTORE SPFILE FROM AUTOBACKUP;

-- Ou depuis un chemin explicite
RESTORE SPFILE FROM AUTOBACKUP
  RECOVERY AREA '/u01/app/oracle/fra'
  DB_NAME 'ORCL';

-- 3. Redemarrer pour utiliser le SPFILE restaure
SHUTDOWN IMMEDIATE;
STARTUP;

Commandes de maintenance et de verification

Verifier l'integrite des sauvegardes 
-- Verifier les sauvegardes (detecte les blocs corrompus)
RESTORE DATABASE VALIDATE;
RESTORE TABLESPACE users VALIDATE;

-- Verifier un fichier specifique
RESTORE DATAFILE 4 VALIDATE;

-- Verifier les archived logs
RESTORE ARCHIVELOG ALL VALIDATE;
Recuperer depuis un emplacement different 
-- Specifier un emplacement de sortie different
RUN {
  SET NEWNAME FOR DATABASE TO '/newpath/%b';
  RESTORE DATABASE;
  SWITCH DATAFILE ALL;
  RECOVER DATABASE;
}

-- Ou pour un datafile specifique
RUN {
  SET NEWNAME FOR DATAFILE '/oradata/users01.dbf' TO '/newdisk/users01.dbf';
  RESTORE DATAFILE '/oradata/users01.dbf';
  SWITCH DATAFILE ALL;
  RECOVER DATAFILE '/newdisk/users01.dbf';
}
Consulter l'historique des operations 
-- Historique des jobs RMAN
SELECT * FROM v$rman_backup_job_details ORDER BY start_time DESC;

-- Liste des sauvegardes disponibles
LIST BACKUP;
LIST BACKUP SUMMARY;
LIST BACKUP OF DATABASE;
LIST BACKUP OF TABLESPACE users;

-- Verifier les archived logs disponibles
LIST ARCHIVELOG ALL;
