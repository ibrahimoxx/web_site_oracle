RMAN — connexion, commandes, configuration, retention

Tout ce que RMAN fait, comment s'y connecter, comment le configurer, et toutes les commandes cles avec leur syntaxe exacte.

Definition Connexion Commandes Configuration Retention

1. Definition et avantages

RMAN (Recovery Manager) est l'outil Oracle integre pour gerer les sauvegardes et recuperations. Il connait la structure interne de la base.

  • Connait les tablespaces, datafiles, control files, archived redo logs, SPFILE.
  • Valide les blocs au moment de la sauvegarde (detecte la corruption).
  • Gere la FRA, les sauvegardes multistream, plusieurs supports de stockage.
  • Supporte les sauvegardes incrementales (seuls les blocs modifies).
  • Trace l'historique complet des operations dans le control file ou le Recovery Catalog.
  • Peut compresser et chiffrer les sauvegardes.

RMAN vs copie manuelle

  • Connait la structure Oracle (pas une copie aveugle).
  • Sauvegarde incrementale: seuls les blocs modifies.
  • Verification de l'integrite des blocs.
  • Historique des operations.
  • Integration avec la FRA.
  • Gestion automatique de la retention.

2. Connexion a RMAN

Le mot-cle TARGET designe la base cible. SYSBACKUP est un ROLE (pas un privilege) recommande pour les operations RMAN: il couvre backup et recovery sans donner acces aux donnees applicatives.

Toutes les formes de connexion 
-- Connexion locale en OS authentication (le plus courant en TP)
rman TARGET /

-- Connexion locale avec le role SYSBACKUP (recommande)
rman TARGET "/ AS SYSBACKUP"

-- Connexion distante avec mot de passe
rman TARGET sys/password@mydb

-- Connexion depuis l'invite RMAN (apres lancement sans argument)
rman
CONNECT TARGET /
CONNECT TARGET sys/password@mydb;

-- Connexion avec Recovery Catalog en plus
rman TARGET / CATALOG rcat_owner/password@catdb
Exam: SYSBACKUP est un ROLE Oracle, pas un privilege systeme. Il est prefere au role DBA pour RMAN car il respecte le principe du moindre privilege: l'administrateur RMAN n'a pas besoin d'acceder aux donnees applicatives.

3. Types de commandes RMAN

TypeDefinitionExemples
Commande autonomeExecutee directement, hors bloc RUN.SHOW ALL, LIST BACKUP, BACKUP DATABASE
Commande de jobExecutee dans un bloc RUN { ... }.ALLOCATE CHANNEL, SET UNTIL
Script RMANEnsemble de commandes sauvegardees (dans control file ou catalog).CREATE SCRIPT, EXECUTE SCRIPT
Bloc RUN — exemple complet 
RUN {
  ALLOCATE CHANNEL c1 DEVICE TYPE DISK FORMAT '/backup/%U';
  ALLOCATE CHANNEL c2 DEVICE TYPE DISK FORMAT '/backup/%U';
  BACKUP AS BACKUPSET DATABASE;
  SQL 'ALTER SYSTEM ARCHIVE LOG CURRENT';
  RELEASE CHANNEL c1;
  RELEASE CHANNEL c2;
}
Scripts RMAN stockes 
-- Creer un script dans le control file ou catalog
CREATE SCRIPT backup_quotidien {
  BACKUP DATABASE PLUS ARCHIVELOG;
  DELETE OBSOLETE;
}

-- Executer le script
EXECUTE SCRIPT backup_quotidien;

-- Executer un script depuis un fichier externe
@/chemin/vers/mon_script.rman

-- Lister les scripts disponibles
LIST SCRIPT NAMES;

-- Supprimer un script
DELETE SCRIPT backup_quotidien;
SQL depuis RMAN 
-- Executer du SQL depuis RMAN
SQL 'ALTER SYSTEM ARCHIVE LOG CURRENT';
SQL 'SELECT name FROM v$datafile';

-- Requete directe (certaines versions)
SELECT name FROM v$datafile;

4. Configuration persistante RMAN

Les parametres CONFIGURE sont persistants: ils survivent a la deconnexion et restent actifs pour toutes les operations futures.

Afficher la configuration 
-- Afficher tous les parametres RMAN
SHOW ALL;

-- Voir les parametres explicitement configures (differents du defaut)
SELECT name, value FROM v$rman_configuration;
Toutes les commandes CONFIGURE importantes 
-- Sauvegarde automatique du control file apres chaque backup
CONFIGURE CONTROLFILE AUTOBACKUP ON;
CONFIGURE CONTROLFILE AUTOBACKUP OFF;

-- Format de l'autobackup
CONFIGURE CONTROLFILE AUTOBACKUP FORMAT FOR DEVICE TYPE DISK TO '/backup/cf_%F';

-- Type de device par defaut
CONFIGURE DEFAULT DEVICE TYPE TO DISK;
CONFIGURE DEFAULT DEVICE TYPE TO SBT;

-- Taille maximale d'un backup set
CONFIGURE MAXSETSIZE TO 100G;

-- Optimisation (evite de sauvegarder des fichiers deja sauvegardes identiques)
CONFIGURE BACKUP OPTIMIZATION ON;

-- Nombre de canaux par defaut
CONFIGURE DEVICE TYPE DISK PARALLELISM 2 BACKUP TYPE TO BACKUPSET;

-- Remettre un parametre a sa valeur par defaut
CONFIGURE RETENTION POLICY CLEAR;
CONFIGURE CONTROLFILE AUTOBACKUP CLEAR;

5. Politique de retention

La retention policy determine quelles sauvegardes sont encore necessaires (non obsoletes). RMAN peut ensuite nettoyer les sauvegardes obsoletes.

PolitiqueSyntaxeSignification
Fenetre de recuperationRECOVERY WINDOW OF n DAYSConserver les sauvegardes necessaires pour recuperer jusqu'a n jours en arriere.
RedondanceREDUNDANCY nConserver exactement n copies de chaque fichier sauvegarde.
Configurer la retention 
-- Fenetre de recuperation de 7 jours
CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 7 DAYS;

-- Garder 2 copies de chaque sauvegarde
CONFIGURE RETENTION POLICY TO REDUNDANCY 2;

-- Desactiver la retention (aucune sauvegarde ne devient obsolete automatiquement)
CONFIGURE RETENTION POLICY TO NONE;
Gerer les sauvegardes obsoletes 
-- Voir quelles sauvegardes sont obsoletes selon la politique
REPORT OBSOLETE;

-- Supprimer les sauvegardes obsoletes
DELETE OBSOLETE;

-- Lister toutes les sauvegardes
LIST BACKUP;
LIST BACKUP SUMMARY;

-- Lister les copies image
LIST COPY;
