Resume TP securite Oracle

Cette page ne resume pas le cours theorique. Elle condense uniquement les operations pratiques vues dans le TP sur les utilisateurs, les privileges, les roles et les mots de passe.

1. Creer et configurer des utilisateurs

  • Creer un utilisateur avec mot de passe et tablespace par defaut.
  • Verifier qu’aucun objet parasite n’est cree dans le tablespace SYSTEM.
  • Attribuer un quota sur le tablespace cible.
  • Donner les privileges minimums de connexion et de creation.

2. Verification via dictionnaire

  • Consulter DBA_USERS pour l’etat des comptes.
  • Consulter DBA_OBJECTS, DBA_SEGMENTS et vues d’espace libre.
  • Verifier quotas et tablespaces associes a l’utilisateur.

3. Privileges et donnees

  • Donner CREATE SESSION et CREATE TABLE.
  • Copier une table du schema SCOTT dans un autre schema.
  • Accorder a un autre utilisateur le droit de lire une table precise.
  • Tester le cas WITH GRANT OPTION.

4. Mots de passe et maintenance

  • Changer un mot de passe via ALTER USER.
  • Expirer le mot de passe pour forcer un changement a la prochaine connexion.
  • Supprimer un quota ou supprimer un utilisateur si necessaire.

Ce que le TP t’apprend vraiment

Passer de la theorie de securite aux commandes DBA concretes: creation d’identites Oracle, controle de l’espace, verification des vues de dictionnaire et distribution pratique des droits.
