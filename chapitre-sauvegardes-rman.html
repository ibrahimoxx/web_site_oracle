Strategies et types de sauvegardes avec RMAN

Types de backup, formes, commandes BACKUP completes, compression, chiffrement — tout le cours avec la syntaxe exacte.

Types de backup Commandes BACKUP Compression Chiffrement Maintenance

1. Types et formats de sauvegarde

TypeDefinitionUsage
Full backupSauvegarde tous les blocs utilises des fichiers vises. N'est pas une base pour les incrementaux.Sauvegarde complete independante.
Incremental level 0Sauvegarde tous les blocs utilises. Sert de base de reference pour les sauvegardes incrementales.Point de depart de la strategie incrementale.
Incremental level 1 differentielleSauvegarde les blocs modifies depuis le dernier incremental (level 0 ou level 1).Restore: rejoue level 0 + tous les level 1 differentiels.
Incremental level 1 cumulativeSauvegarde les blocs modifies depuis le dernier level 0.Restore: rejoue level 0 + dernier level 1 cumulatif seulement.
Image copyCopie physique exacte des fichiers (datafiles, archived logs). Restauration plus rapide.Quand la vitesse de restore prime sur l'espace.
Backup setFormat RMAN compresse et compact. Moins d'espace mais restauration plus lente.Quand l'espace de stockage est contraint.
Proxy copyDelegue la sauvegarde a un outil externe via un MML (Media Management Library).Environnements avec librairies de bandes tierces.
Exam — difference cle: Incremental level 1 differentielle = depuis le dernier incremental (level 0 OU level 1). Incremental level 1 cumulative = toujours depuis le dernier level 0. La cumulative necessite moins de sauvegardes pour le restore mais est plus volumineuse.

2. Commandes BACKUP — syntaxe complete

Sauvegardes de la base entiere 
-- Sauvegarde complete de la base
BACKUP DATABASE;

-- Sauvegarde avec les archived redo logs (recommande)
BACKUP DATABASE PLUS ARCHIVELOG;

-- Ignorer les tablespaces en lecture seule
BACKUP DATABASE SKIP READONLY;

-- Voir la structure de la base avant de sauvegarder
REPORT SCHEMA;
Sauvegardes partielles 
-- Sauvegarde d'un tablespace specifique
BACKUP TABLESPACE users;
BACKUP TABLESPACE users, example;

-- Sauvegarde d'un datafile specifique
BACKUP DATAFILE 4;
BACKUP DATAFILE '/oradata/users_01.dbf';

-- Sauvegarde des archived redo logs seulement
BACKUP ARCHIVELOG ALL;
BACKUP ARCHIVELOG FROM SCN 100000 UNTIL SCN 200000;
Sauvegardes CDB/PDB (multitenant) 
-- Sauvegarde d'une PDB specifique
BACKUP PLUGGABLE DATABASE hr_pdb;

-- Sauvegarde du CDB$ROOT
BACKUP PLUGGABLE DATABASE "CDB$ROOT";

-- Sauvegarde de toutes les PDB + CDB
BACKUP DATABASE;
Sauvegardes incrementales 
-- Level 0 : base de reference (tous les blocs utilises)
BACKUP INCREMENTAL LEVEL 0 DATABASE;

-- Level 1 differentielle (depuis le dernier incremental, level 0 ou 1)
BACKUP INCREMENTAL LEVEL 1 DATABASE;

-- Level 1 cumulative (depuis le dernier level 0)
BACKUP INCREMENTAL LEVEL 1 CUMULATIVE DATABASE;
Image copies 
-- Copie image d'un datafile
BACKUP AS COPY DATAFILE '/oradata/users_01_db01.dbf';

-- Copie image de tous les archived logs correspondant a un motif
BACKUP AS COPY ARCHIVELOG LIKE '/arch%';

-- Copie image de la base entiere
BACKUP AS COPY DATABASE;
Backup sets avec format 
-- Backup set simple
BACKUP AS BACKUPSET DATABASE;

-- Backup set avec format de nommage personnalise
BACKUP AS BACKUPSET
  FORMAT '/backup/df_%d_%s_%p.bkp'
  TABLESPACE hr_data;

-- Backup set avec format et plus archivelog
BACKUP AS BACKUPSET DATABASE
  FORMAT '/backup/%d_%T_%U'
  PLUS ARCHIVELOG;
Dans un bloc RUN avec canal dedie 
RUN {
  ALLOCATE CHANNEL c1 DEVICE TYPE DISK FORMAT '/backup/%U';
  BACKUP DATABASE PLUS ARCHIVELOG;
  BACKUP CURRENT CONTROLFILE;
  DELETE OBSOLETE;
}

3. Compression des sauvegardes

La compression reduit l'espace occupe par les sauvegardes au prix d'une consommation CPU supplementaire. Plus l'algorithme est puissant, plus il compresse mais plus il consomme de CPU.

AlgorithmeCompressionCPUUsage recommande
BASICFaibleFaibleDefaut, bon equilibre.
LOWBasseTres faibleSauvegarde rapide, peu de CPU disponible.
MEDIUMMoyenneMoyenBon equilibre espace/CPU.
HIGHEleveeEleveEspace critique, CPU disponible.
Configurer et utiliser la compression 
-- Configurer l'algorithme par defaut (persistant)
CONFIGURE COMPRESSION ALGORITHM 'BASIC';
CONFIGURE COMPRESSION ALGORITHM 'LOW';
CONFIGURE COMPRESSION ALGORITHM 'MEDIUM';
CONFIGURE COMPRESSION ALGORITHM 'HIGH';

-- Sauvegarde compressee avec l'algorithme configure
BACKUP AS COMPRESSED BACKUPSET DATABASE PLUS ARCHIVELOG;

-- Sauvegarde compressee avec algorithme specifie dans le bloc RUN
RUN {
  SET COMPRESSION ALGORITHM 'HIGH';
  BACKUP AS COMPRESSED BACKUPSET DATABASE PLUS ARCHIVELOG;
}

4. Chiffrement des sauvegardes

Le chiffrement protege la confidentialite des sauvegardes. Indispensable si les supports de sauvegarde quittent les locaux ou sont stockes dans le cloud.

Chiffrement avec mot de passe 
-- Chiffrer la sauvegarde avec un mot de passe
SET ENCRYPTION IDENTIFIED BY 'mon_mot_de_passe' ONLY;
BACKUP DATABASE PLUS ARCHIVELOG;

-- Lors de la restauration, fournir le mot de passe de dechiffrement
SET DECRYPTION IDENTIFIED BY 'mon_mot_de_passe';
RESTORE DATABASE;
RECOVER DATABASE;
Important: Sans le mot de passe de chiffrement, une sauvegarde chiffree est inutilisable. Stocker le mot de passe en lieu sur, separe des sauvegardes elles-memes.

5. Maintenance des sauvegardes

Lister et verifier 
-- Lister toutes les sauvegardes
LIST BACKUP;
LIST BACKUP SUMMARY;

-- Lister les copies image
LIST COPY;

-- Verifier que les sauvegardes sont accessibles sur le disque
CROSSCHECK BACKUP;
CROSSCHECK ARCHIVELOG ALL;

-- Verifier une sauvegarde specifique
CROSSCHECK BACKUP OF DATABASE;
Nettoyer les sauvegardes 
-- Supprimer les sauvegardes expiredees (fichiers inaccessibles)
DELETE EXPIRED BACKUP;

-- Supprimer les sauvegardes obsoletes (selon la politique de retention)
DELETE OBSOLETE;

-- Supprimer les archived logs deja sauvegardes au moins une fois
DELETE ARCHIVELOG ALL BACKED UP 1 TIMES TO DISK;

-- Supprimer une sauvegarde specifique
DELETE BACKUPSET 5;
