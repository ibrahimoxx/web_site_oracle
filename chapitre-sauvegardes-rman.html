Strategie et types de sauvegardes avec RMAN

Ici, le cours passe de l’outil RMAN a la conception de vraies sauvegardes: quoi sauvegarder, sous quelle forme, a quelle frequence et avec quelle strategie.

Strategie Types Options Commandes

1. Strategie de sauvegarde

  • Une strategie doit couvrir le besoin metier en disponibilite et en reprise.
  • Elle depend du volume de stockage disponible, du type de base et du temps acceptable de restauration.
  • Elle peut viser toute la base (whole) ou seulement une partie (partial).

2. Types de backup

  • Full backup: tous les blocs utilises des fichiers vises.
  • Incremental level 0: equivalent logique d’une base de reference complete.
  • Incremental level 1 cumulative: changements depuis le dernier niveau 0.
  • Incremental level 1 differentielle: changements depuis le dernier incremental precedent.

3. Modes et formes de sauvegarde

  • Offline: sauvegarde a froid, base arretee, donc coherente.
  • Online: sauvegarde a chaud, base en service.
  • Image copy: copie directe des fichiers, plus lourde en espace mais plus rapide a restaurer.
  • Backup set: format binaire plus compact, mais necessite une reconstruction lors de la reprise.
  • Proxy copy: appui sur un outil externe de sauvegarde.

4. Ce qui peut etre inclus

  • Base complete.
  • PDB ou tablespace specifique.
  • Archived redo logs.
  • Control files et spfile.

5. Options de sauvegarde

  • Compression: moins d’espace, cout de traitement supplementaire.
  • Cryptage: protection de la confidentialite des sauvegardes.
  • Block change tracking: facilite les incrementaux en suivant les blocs modifies.

6. Choix d’une bonne strategie

Le cours insiste sur le fait qu’il n’existe pas une strategie unique optimale. Il faut arbitrer entre rapidite de backup, rapidite de restore, espace consomme et niveau de protection voulu.

7. Commandes du cours a memoriser

Structure et sauvegardes globales

Ce sont les commandes les plus directement rattachees au support.

REPORT SCHEMA;
BACKUP DATABASE;
BACKUP DATABASE PLUS ARCHIVELOG;
BACKUP DATABASE SKIP READONLY;

CDB, PDB et tablespace

Le cours insiste sur la sauvegarde totale ou partielle.

BACKUP PLUGGABLE DATABASE nom_pdb;
BACKUP TABLESPACE users;
BACKUP TABLESPACE nom_pdb:users;

Incremental

Le support parle explicitement du niveau 0 et du tag d'incremental.

BACKUP INCREMENTAL LEVEL 0 DATABASE;
BACKUP INCREMENTAL LEVEL 1 TAG 'daily_inc' DATABASE;

Image copy et backup set

Deux formes de sauvegarde a savoir reconnaitre.

BACKUP AS COPY DATAFILE '/chemin/datafile.dbf';
BACKUP AS COPY ARCHIVELOG LIKE 'arch%';
BACKUP AS BACKUPSET TABLESPACE users;

Control file, compression et chiffrement

Le cours associe ces options a la strategie de protection.

CONFIGURE CONTROLFILE AUTOBACKUP ON;
BACKUP AS COMPRESSED BACKUPSET DATABASE;
SET ENCRYPTION ON IDENTIFIED BY 'motdepasse' ONLY;

Lecture d'examen

Pour ce chapitre, sais relier chaque commande a son but.

  • REPORT SCHEMA pour voir datafiles et tablespaces.
  • BACKUP AS COPY pour image copy.
  • BACKUP AS BACKUPSET pour ensemble compresse/logique.
  • LEVEL 0 / LEVEL 1 pour incremental.

Resume flash

Full, level 0, cumulative, differentielle, image copy et backup set sont des notions centrales. L’examen peut demander la difference theorique, le bon cas d’usage ou le compromis espace/temps.
