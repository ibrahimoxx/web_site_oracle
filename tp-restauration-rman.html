Resume TP restauration et recuperation avec RMAN

Cette page synthetise les scenarios pratiques testes dans le TP: perte d’un control file, perte du spfile et perte d’un datafile critique.

1. Scenario control file

  • Faire une sauvegarde complete avec archivelog.
  • Activer l’autobackup des control files.
  • Perdre un control file, redemarrer en NOMOUNT.
  • Restaurer le control file depuis l’autobackup.
  • Monter, recuperer la base puis ouvrir avec RESETLOGS.

2. Scenario spfile

  • Perdre le spfile.
  • Utiliser le DBID si necessaire.
  • Restaurer le spfile depuis l’autobackup.
  • Redemarrer la base apres restauration.

3. Scenario datafile critique

  • Simuler la perte d’un datafile systeme ou undo.
  • Observer que la base ne peut pas continuer normalement.
  • Restaurer puis recuperer le fichier avec RMAN.

4. Point a memoriser

  • Le mode de demarrage depend de ce qui manque: NOMOUNT, MOUNT, puis OPEN.
  • Le scenario avec control file de sauvegarde se termine souvent par une nouvelle incarnation.
